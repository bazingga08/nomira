# Generation Playbook — Stage 3 (Divergent, Judgment BANNED)

Diverge mile-wide. JUDGMENT IS BANNED in this stage: do not score, reject, or
self-censor any candidate here. Volume is the goal; culling happens later. Hit the
target phonetic profile, cover every morphological process, cover every territory, and
run all three disguised-brief teams.

---

## The three-team disguised-brief method
Run three parallel generation frames. The disguise is what harvests names a single
on-brief team would never reach.
- **Team A — real brief.** Generates directly against the strategy object.
- **Team B — competitor frame.** BELIEVES it is naming a COMPETITOR in the same
  category. Surfaces names the real team would self-censor; you then re-point the
  ownable ones back to the real brief.
- **Team C — unrelated category.** Told it is naming something else entirely (e.g.
  "name a bike", "name a coffee shop"). Harvests lateral, ownable names that can be
  re-pointed. This is the expansionist BlackBerry move (fruit → email device).

---

## Per-team morphological quota (every process, every team)
Each team must produce candidates across ALL eight processes:

| Process | What it is | Examples |
|---|---|---|
| Descriptive compound | two real words that state the function | PayPal, Salesforce |
| Blend / portmanteau | fuse two words/morphemes | Pinterest, Microsoft, Groupon |
| Clipping | shorten a word | FedEx, Cisco, Intel |
| Neoclassical | Latin/Greek combining forms | Pentium, Acura, Verizon |
| Respelling | real word, altered spelling | Flickr, Lyft, Tumblr |
| Pure neologism / sound-symbolic | invented from scratch | Kodak, Dasani, Häagen-Dazs |
| Arbitrary real-word | unrelated real word | Apple, Amazon, Azure |
| Metaphor / evocative | image that implies the promise | Oracle, Jaguar, Windsurf |

---

## Construction techniques
- **Mine the use-case verb.** Swiffer ← swift/sweep/swipe + -er (agentive). Start from
  what the user DOES with the product.
- **Stack morphemes that SUM to the promise.** Pentium = pent- (five, the P5/586) +
  -ium (element suffix → scientific/premium). The decoded sum IS the positioning.
- **Hit the target phonetic profile.** Pull phonemes that carry the hero attribute
  (`sound-symbolism.md`). Reliability → /b d g/ + back vowels; speed → /p t k/ + /i/.
- **Break category convention.** Dasani broke bottled-water place-names; Azure broke
  "cloud X". Map the convention in the competitive audit, then deliberately violate it.
- **Re-point lateral names.** A Team-C "bike" name with the right sound/feel can be
  adopted for the real product.

### Worked recipes
- **Pentium** = pent- (five) + -ium (element) → "a fundamental element of computing";
  /p/ /t/ voiceless stops = fast/sharp; scientific-premium feel.
- **Swiffer** = swift + -er, with /sw/ sound-symbolic sweeping motion → quick/effortless.
- **Verizon** = veritas (truth) + horizon → forward-looking reliability.
- **Febreze** = fabric + breeze, respelled → fresh air; /f/ /z/ fricatives = airy/light.

## Morpheme-construction engine (step by step)
1. List 5–10 ROOTS carrying the core meaning (real-word fragments, Latin/Greek, the
   benefit word).
2. List 5–10 AFFIXES carrying connotation or category signal (-ium, -ly, -o, -a, -ex,
   -ify, ver-, -cel, -san, -ova).
3. CROSS them (a 6×6 grid = 36 raw combos). Keep pronounceable ones that yield a third,
   emergent meaning ("1+1=3").
4. Tune phonemes toward the target profile (back vowel in for "premium", voiceless stop
   in for "fast").
5. Read each aloud — kill tongue-twisters and accidental bad words now.

## "Relevant, but unexpected" — the distortion move
Don't stop at the obvious literal phrase. Take the on-strategy concept and distort it
into a distinctive coined form: "fresh breeze" → Febreze. Pipeline: literal phrase →
clip/blend/respell → phoneme-tune → check it still decodes. The single most productive
technique for names that are both meaningful and ownable.

## Note on AI-product names (2026)
Avoid anthropomorphic/human names (prevents overpromising) and overly technical terms
(Codeium → Windsurf). Trademark saturation makes coined/arbitrary names the safer bias.

## Volume targets per scale_mode
- **lite** (feature / rename): ~60 raw candidates.
- **standard** (product): ~120–200 raw candidates.
- **full** (company / brand): 200–600+ raw candidates.
- For context, Lexicon itself generates 1,000–3,000 raw candidates and eliminates ~90%;
  an AI can push toward that ceiling at near-zero marginal cost — loop Stage 3↔6 around
  winning veins rather than settling for the first pool.

## Territory coverage & GATE
Within each team, generate across ALL naming territories from the strategy object. Aim
for at least one candidate in every (team × process × territory) cell.

**GATE:** minimum candidate count met AND every morphological bucket non-empty AND all
three team-frames non-empty. Otherwise regenerate ONLY the empty buckets — do not
restart the whole pool.
