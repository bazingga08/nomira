# Intake Questions — The Brief-Interrogation Battery

The brief is half the battle. Refuse to generate before the strategy object exists —
but NEVER block on a thin brief. Ask AT MOST the top 5 unfilled questions in ONE
batch, highest-leverage first. For every slot left unanswered, INFER a category-norm
default and STATE it explicitly in the assumptions block.

---

## Tier 1 — Strategy-critical (always needed; answered or inferred)

**(a) WHAT IS IT, in one sentence?**
The product/company/feature and the core function it performs.

**(b) WHO is the primary audience, and what is the BUYER CONDITION?**
Impulse / low-consideration consumer vs deliberate / high-consideration B2B. This
sets how heavily to weight sound symbolism (Yorkston-Menon; Alter-Oppenheimer):
heavy for impulse consumers, lighter for deliberate B2B buyers.

**(c) What is the SINGLE hero attribute / brand promise the name must carry?**
The Lowrey-Shrum match rule needs EXACTLY ONE. If the user lists several, force a
ranking and pick the top one.

**(d) What future customer BEHAVIOR or FEELING should the name create — and is there
a downside/anxiety to counter-position against?**
e.g. BlackBerry counter-positioned against email/tech anxiety.

**(e) What PERSONALITY should it project — pick ONE Jennifer Aaker dimension?**
Sincerity, Excitement, Competence, Sophistication, or Ruggedness.

**(f) HARD CONSTRAINTS?**
Length/syllables, must-include sounds/letters, must-AVOID sounds/words, target
markets + languages to clear, relevant Nice trademark class(es), and whether a
matching domain/handle is required.

## Tier 2 — Resolve only if cheap (else infer from the competitive audit)

**(g) Key COMPETITORS and their phonetic/morphological texture?** Used to find white
space. If not cheap to get, infer from the Stage-1 competitive audit.

**(h) Names you already LOVE or HATE, and why?** Calibrates taste WITHOUT overriding
strategy.

**(i) Any FOUNDER, heritage, mission, or origin references** that should or should
not be reflected?

---

## RULE
Ask at most the top 5 unfilled questions in ONE batch, highest-leverage first. For
every unanswered slot, INFER a category-norm default and STATE it explicitly in the
assumptions block. Never block a thin brief.

---

## Default-inference rules (state every default you use)

| Slot | Default when unanswered |
|---|---|
| Target market | global, English-first |
| Syllables / length | 2–3 syllables |
| Personality | Competence (safest neutral default) |
| Buyer condition | infer from category (SaaS → deliberate B2B; consumer app/CPG → impulse) |
| Phonetic profile | derive from the hero attribute via `sound-symbolism.md` |
| Hero attribute | if truly none given, infer the dominant category promise and STATE it as a guess to confirm |
| Domain requirement | assume "nice-to-have, not a veto" unless told otherwise |
| Nice class | infer from what-it-is; flag as "assumed, confirm before TM search" |

State inferences like: *"Assuming global English-first market, 2–3 syllables,
Competence personality since unspecified, and 'reliability' as the hero attribute —
correct any of these and I'll rerun the strategy."*

## WHEN-NOT-TO-RENAME guard (run only if `rename_flag`)

Renaming destroys accumulated brand equity; the bar is high. Require ALL of:
1. A real **business catalyst** — merger/acquisition, repositioning, legal forcing
   function, market shift, or a name actively blocking growth.
2. A clear **customer benefit** — the new name helps the customer, not just internal ego.
3. **Internal consensus** — stakeholders are aligned that renaming is worth it.

If stakeholders are SPLIT, advise AGAINST renaming and STOP. A divided org will not
support the equity rebuild a rename demands. Offer alternatives first: a tagline
shift, sub-brand, or repositioning of the existing name.
